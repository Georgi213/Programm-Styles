Programming Style

RUS
Behavior-driven development

Behavioral development is a method of software development that has evolved from test-based development. Behavioral development combines the general techniques and principles of testing-based development. It aims to provide a shared tool for software development to the development team and management team.

Design-driven development

Design driven development uses design as part of a process to learn and better define requirements in order to build better, more informed technology solutions. It can also be looked at as a process whereby design and user experience drives the development of a product or software application. This leads to products that people enjoy using and want to tell others about.

Domain-driver design

Domain-based design is a software design approach that focuses on modeling software to fit a domain according to the input of experts in that domain. In object-oriented programming, this means that the structure and language of the software code should match the business area.

Secure by design

Secure by design, in software engineering, means that software products and capabilities have been designed to be foundationally secure. Alternate security strategies, tactics and patterns are considered at the beginning of a software design, and the best are selected and enforced by the architecture, and they are used as guiding principles for developers. It is also encouraged to use strategic design patterns that have beneficial effects on security, even though those design patterns were not originally devised with security in mind.

Test-driven development

Test-based development is a method of software development where tests are written before the software itself. The development process consists of short iterations, in which the failed test is written first and then the minimum code to pass the test. This development ensures that all code is always tested.

Acceptance test-driven develpment

Acceptance-based development is a development methodology based on communication between business customers, developers and testers. ATDD encompasses many of the same practices as specification specification, behavioral development, role-based development, and support-based development, also known as story-based development.

Continuous test-driven development

In CTDD the developer writes a test first but is not forced to execute the tests manually. The tests are run automatically by a continuous testing tool running in the background. This technique can potentially reduce the time waste resulting from manual test execution by eliminating the need for the developer to start the test after each phase of the normal TDD practice: after writing the (initially failing) test, after producing the minimal amount of code for the test to pass and after refactoring the code.

Specification by example

A sample specification is a collaborative approach to defining software product requirements and business-oriented functional tests based on capturing and illustrating requirements using realistic examples instead of abstract statements.

Data-driven development

In computer programs, data-based programming is a programming paradigm in which program statements describe the data to be matched and the processing required, rather than defining a sequence of steps to take.

Data-oriented design

It is a data-driven design program optimization approach that is motivated by the efficient use of the CPU cache used in video game development.
